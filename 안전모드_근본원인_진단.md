# 🔍 Elementor 안전 모드 근본 원인 진단

## 🎯 모든 일반적인 해결책을 시도했는데도 안전 모드가 계속되는 이유

### 가능한 원인 7가지:

1. **플러그인 충돌** (가장 흔함 - 60%)
2. **테마 충돌** (20%)
3. **Elementor 버전 버그** (10%)
4. **서버 환경 문제** (PHP, 메모리)
5. **JavaScript 파일 손상**
6. **브라우저 확장 프로그램**
7. **wp-config.php 설정 충돌**

---

## ⚡ 체계적 진단 프로세스

### 🔥 1단계: 플러그인 충돌 확인 (가장 중요!)

#### 방법 A: WordPress 관리자에서

```
1. 플러그인 → 설치된 플러그인
2. Elementor와 Elementor Pro를 제외한 모든 플러그인 비활성화
3. Elementor 편집기 다시 열기
4. 안전 모드가 사라졌나요?
```

**사라졌다면:**
- 플러그인을 하나씩 다시 활성화
- 안전 모드가 다시 나타나는 순간의 플러그인이 범인

**여전히 안전 모드라면:**
- 2단계로 이동

#### 방법 B: phpMyAdmin으로 일시 비활성화

```sql
-- 모든 플러그인 비활성화 (Elementor 제외)
UPDATE wp_options 
SET option_value = 'a:2:{i:0;s:33:"elementor/elementor.php";i:1;s:37:"elementor-pro/elementor-pro.php";}' 
WHERE option_name = 'active_plugins';
```

---

### 🔥 2단계: 테마 충돌 확인

```
1. 외모 → 테마
2. "Hello Elementor" 테마로 변경
   (없다면 설치: 외모 → 테마 → 새로 추가 → "Hello Elementor")
3. Elementor 편집기 다시 열기
4. 안전 모드가 사라졌나요?
```

**사라졌다면:**
- 현재 사용 중인 테마가 원인
- 테마 제작자에게 문의 또는 다른 테마 사용

---

### 🔥 3단계: Elementor 버전 확인

#### 현재 버전 확인:

```
플러그인 → 설치된 플러그인
→ Elementor 버전 확인
→ Elementor Pro 버전 확인
```

#### 알려진 버그 버전:

- **Elementor 3.11.x** - 일부 버전에서 안전 모드 버그
- **Elementor 3.16.x** - 초기 버전 안정성 문제
- **Elementor Pro 3.11.x** - 라이센스 관련 안전 모드 버그

#### 해결 방법:

**A) 최신 버전으로 업데이트:**
```
대시보드 → 업데이트
→ Elementor 업데이트 확인
```

**B) 이전 버전으로 다운그레이드:**
```
1. WP Rollback 플러그인 설치
2. Elementor를 3.21.x (안정 버전)로 롤백
```

---

### 🔥 4단계: 서버 환경 확인

#### PHP 버전 확인:

```
도구 → 사이트 상태 → 정보
→ PHP 버전 확인
```

**권장:**
- PHP 7.4 이상
- PHP 8.0 - 8.2 (최적)

**PHP 8.3+는 호환성 문제 가능**

#### 메모리 제한 확인:

```
도구 → 사이트 상태
→ "PHP 메모리 제한" 확인
```

**권장:** 최소 256MB

**부족하면 wp-config.php에 추가:**
```php
define('WP_MEMORY_LIMIT', '512M');
define('WP_MAX_MEMORY_LIMIT', '512M');
```

#### Cloudways 서버 재시작:

```
Cloudways 패널
→ Manage Services
→ PHP-FPM → Restart
→ Nginx → Restart
```

---

### 🔥 5단계: JavaScript 파일 손상 확인

#### Elementor 파일 재생성:

**phpMyAdmin에서:**
```sql
-- Elementor 캐시 완전 삭제
DELETE FROM wp_options WHERE option_name LIKE 'elementor_css_print_method%';
DELETE FROM wp_options WHERE option_name LIKE '_elementor_global_css%';
DELETE FROM wp_options WHERE option_name LIKE 'elementor_optimized_assets%';
```

#### SFTP/파일 관리자로:

```
/wp-content/uploads/elementor/css/ → 전체 삭제
/wp-content/cache/elementor/ → 전체 삭제 (있다면)
```

---

### 🔥 6단계: 브라우저 확장 프로그램 확인

#### 시크릿 모드 + 모든 확장 비활성화:

```
Chrome:
1. Ctrl + Shift + N (시크릿 모드)
2. 주소창 오른쪽 "확장 프로그램" 아이콘
3. "시크릿 모드에서 허용" 모두 끄기
4. Elementor 편집기 열기
```

**의심되는 확장:**
- 광고 차단기 (uBlock Origin, AdBlock 등)
- 보안 프로그램 (Kaspersky, Avast 등)
- VPN
- 개발자 도구 (React DevTools 등)

---

### 🔥 7단계: wp-config.php 설정 확인

#### 디버그 모드 확인:

**wp-config.php에서 이 줄들 확인:**

```php
// 디버그 모드가 켜져 있으면 끄기
define('WP_DEBUG', false);
define('WP_DEBUG_DISPLAY', false);
define('SCRIPT_DEBUG', false);

// 자동 업데이트 비활성화 제거
// define('AUTOMATIC_UPDATER_DISABLED', true); // 이 줄이 있으면 주석 처리
```

## 🔥 8단계: Elementor 완전 재설치 (최종 수단)

### 설정 백업:

```
Elementor → 도구 → 버전 관리
→ 현재 설정 백업 다운로드
```

### 재설치 과정:

```
1. Elementor Pro 비활성화 → 삭제
2. Elementor 비활성화 → 삭제
3. phpMyAdmin에서 완전 클린:
   DELETE FROM wp_options WHERE option_name LIKE 'elementor%';
   DELETE FROM wp_usermeta WHERE meta_key LIKE 'elementor%';
4. SFTP에서 폴더 삭제:
   /wp-content/uploads/elementor/
5. Elementor 재설치
6. Elementor Pro 재설치 + 라이센스 입력
```

## 🆘 가장 흔한 원인 TOP 3

### 1. 플러그인 충돌 (60%)

**흔한 범인 플러그인:**
- 캐시 플러그인 (WP Rocket, W3 Total Cache 등)
- 보안 플러그인 (Wordfence, Sucuri 등)
- 최적화 플러그인 (Autoptimize, Asset CleanUp 등)
- 번역 플러그인 (WPML, Polylang 등)
- 페이지 빌더 플러그인 (다른 빌더와 충돌)

### 2. 테마 충돌 (20%)

**문제가 있는 테마:**
- 구버전 테마 (3년 이상 업데이트 안 됨)
- 불법 복제 테마
- 과도한 기능이 있는 테마

**해결:**
- Hello Elementor 또는 Astra 사용 권장

### 3. Elementor 버전 버그 (10%)

**해결:**
- 최신 안정 버전으로 업데이트
- 또는 3.21.x로 다운그레이드
